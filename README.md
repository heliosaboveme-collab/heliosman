１つ目のシューティングゲームについて
これはjavaのswing用の非常に初歩的な構築ができることを示すための作品です。大学一年生のときに講義で基礎を学んで作成いたしました。
ソースコードで力を入れたのは自機のマウスポイント機能と敵機及びその弾の動きの物理演算です。
基本的に仮想windowを構築しない限りgithubでは実行できませんので実際に使用される場合は、デスクトップにフォルダごと保存し、エクスプローラー（PC内のアプリ）でそのフォルダを開いた状態でコマンドプロンプトを開いて
コンパイルと実行を押して頂けるとプレイできます。

２つ目のブラックジャックについて
java scriptを用いたwebゲームです。
合計６時間程度で制作しました。ソースコードについては、それまでhtmlを用いたUIの制作は未経験でしたので、AIに教えてもらいながらの共同制作でしたが苦戦しました。通常のブラックジャックでのAが1にも11にも変化するという点や、透視などのイカサマルール、対戦相手のディーラーの追加カードを引く条件（17以上になるように引く）などは難易度も考慮して良い塩梅を考え尽くしました。
改善したい点は、PC本体のダークモードに依存した白の反転です。トランプのカードの白の割合が多いと、chromeの判定で色の逆転が起こりますので、そちらを対処することが現在の目標です。
こちらはgithubページの右下にあるgo liveを押した後、Blackjack=JSverをクリックしていただけますとプレイできます。

３つ目の誹謗中傷保全ツールについて
こちらは現在開発中の自信作の途中経過になります。githubではなくgoogle AI studioで制作したAIを用いた誹謗中傷コメントの自動取得ツールです。そのためgithub上では機能しません。
機能自体の概要といたしましては、スプレッドシートに貼り付けたyoutubeの動画を読み取り、youtubeAPIで新着コメントを指定数読み取ってそのコメントが危険度の高いものかをAIが判断し、証拠として必要な情報を出力シートに書き込む、というものです。
開発において力を入れた点は２つございます。１つ目は処理速度をあげるための差分更新です。新着コメントの中に前回の読み取ったコメントIDと同様のIDを持ったコメントはスルーできる様になっています。
２つ目は、コスト削減のためのプロンプトの工夫です。判定AIは従課金制で、入力と出力（特に出力）の文字数でかかる金額が決まります。そのため危険度の低いコメントの判定結果まで出力させると費用倒れになりますので、低いものにはCとだけ出力するようになっています。
今後の改善点といたしましては、スクリーンショットのAPIの追加を目標としています。youtubeの仕様上、bot判定されるとコメントが表示されませんので、自動でスクロールするというAPIが使えない状況です、リンクのみでそのコメントのページに飛び、スクリーンショットを撮らせることで証拠の強化を図りたい所存です。
